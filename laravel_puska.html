4 Factory & Faker — Véletlenszerű adatok

1. Magyar Faker: nyisd meg a config/app.php fájlt és írd át:

'faker_locale' => 'hu_HU',

2. A factory fájl (database/factories/KimenohivasokFactory.php):

public function definition(): array
{
    return [
        // Szöveg / Név
        'nev'        => $this->faker->name(),
        'telszam'    => $this->faker->phoneNumber(),

        // Dátum INTERVALLUM között:
        'nap'        => $this->faker->dateTimeBetween('-2 weeks', '+1 day')->format('Y-m-d'),

        // Szám MIN és MAX között:
        'ora'        => $this->faker->numberBetween(8, 19),
        'perc'       => $this->faker->numberBetween(0, 59),
        'masodperc'  => $this->faker->numberBetween(0, 59),

        // Egész szám:
        'hossz'      => $this->faker->numberBetween(30, 3600),

        // Logikai (igaz/hamis véletlenszerűen):
        'visszahivas' => $this->faker->boolean(),
    ];
}

Adatok importálása CSV fájlból (Gyakori vizsgafeladat!)

Ha kapsz egy adatok.csv fájlt, másold a database/seeders mappába. Példa egy beolvasó Seederre:

public function run(): void
{
    $file = database_path('seeders/adatok.csv');
    $handle = fopen($file, 'r');

    // 1. Fejléc beolvasása (Figyelj az elválasztóra! Lehet ';' vagy ',')
    $headers = fgetcsv($handle, 0, ';'); 
    
    // Tipp: Ha a fejlécek idézőjelben vannak ("id", "nev"), így szedd ki:
    $headers = array_map(fn($h) => trim($h, '"'), $headers);

    // 2. Adatsorok beolvasása
    while (($row = fgetcsv($handle, 0, ';')) !== false) {
        $data = array_combine($headers, $row);
        
        SajatModell::create([
            // A CSV oszlopai alapján hivatkozz rájuk:
            'perc'       => (int) $data['perc'],
            'cim'        => trim($data['cim']),
            // Ha a CSV-ben NULL szöveg van:
            'megjegyzes' => ($data['megjegyzes'] !== 'NULL') ? trim($data['megjegyzes']) : null,
        ]);
    }

    fclose($handle);
}

⚠️ Tipikus CSV Import hibák:

  • Undefined array key "valami": Vagy az elválasztó rossz (; helyett ,), vagy a fejlécben idézőjelek vannak és azt is beolvasta. Használj var_dump($headers)-t debugoláshoz!
  • Failed to open stream: Nincs ott a fájl! Másold be a database/seeders mappába.
  • Oszlopnév elírások: A CSV oszlopnevei (pl. mperc) nem mindig egyeznek meg a te adatbázis oszlopaiddal (pl. mpercg). A Seederben így párosítsd: 'mpercg' => $data['mperc']

10 Gyors tippek & Buktatók

🕐 Időpont — csak 0-ra végződő percek

Az input formon step="600" beállítással a böngésző csak 10 perces lépéseket enged (pl. 13:00, 13:10, 13:20...). A feladat minden esetben elfogadja ezt a megoldást.

☑️ Checkbox bug mentésnél

Ha a checkbox nincs bepipálva, a böngésző el sem küldi azt a mezőt. A store() és update() metódusban kézzel kell beállítani: $request->merge(['visszahivas' => 0])

🔑 $fillable vs $guarded

Ha a feladat sok mezőt kér, egyszerűbb a protected $guarded = []; beállítás a modellben — ez mindent engedélyez. Vizsgán elfogadható.

🧠 @auth vs middleware

@auth Blade direktíva: csak a nézet egy részét rejti el. A ->middleware('auth') a route-on: teljesen megtiltja az oldal elérését ha nem vagy belépve.

📦 groupBy() a controllerben

Ha a feladat "mérési hely szerinti bontást" kér: ->get()->groupBy('meresi_hely'). A Blade-ben akkor két egymásba ágyazott @foreach kell.

11 Validáció — Adatok ellenőrzése (store() / update())

Ha a feladat azt kéri, hogy adatok megadása előtt ellenőrizzük a mezőket, a $request->validate() a megoldás. Hiba esetén automatikusan visszairányít az űrlapra.

Alap validáció (store metódus)

public function store(Request $request)
{
    $validated = $request->validate([
        // Kötelező szöveg, egyedi az egész táblában:
        'tanterem'           => 'required|string|unique:termeks,tanterem',
        // Kötelező egész szám, minimum értékkel:
        'befogadokepesseg'   => 'required|integer|min:1',
        // Nem kötelező, de ha van, legyen egész:
        'tv_meret'           => 'nullable|integer|min:1',
        // Logikai mező:
        'projektor'          => 'boolean',
        // Egyéni üzleti logika (closure szintaxis):
        'berbeadas_osszege'  => [
            'required', 'integer', 'min:0',
            function ($attribute, $value, $fail) {
                if ($value % 1000 !== 0 && $value % 500 !== 0) {
                    $fail('Az összegnek 500-ra vagy 1000-re kell végződnie!');
                }
            },
        ],
    ], [
        // Saját hibaüzenetek:
        'tanterem.unique'  => 'Ez a teremszám már létezik!',
        'tanterem.required'=> 'A terem megadása kötelező!',
    ]);

    Termek::create($validated); // create() a validált adatokkal
    return redirect()->route('termekek.index')->with('success', 'Sikeresen létrehozva!');
}

Módosításnál: unique az éppen szerkesztett sort kihagyva (!):

use Illuminate\Validation\Rule;

public function update(Request $request, $id)
{
    $termek = Termek::findOrFail($id);

    $validated = $request->validate([
        // Az 'unique' rule-t kiegészítjük: hagyja figyelmen kívül az éppen szerkesztett sort!
        'tanterem' => [
            'required', 'string',
            Rule::unique('termeks', 'tanterem')->ignore($termek->id)
        ],
    ]);

    $termek->update($validated);
    return redirect()->route('termekek.index')->with('success', 'Sikeresen frissítve!');
}

A leggyakoribb validációs szabályok táblázata:

Szabály Mit csinál
required Kötelező — nem lehet üres
nullable Nem kötelező, de ha van, ellenőrzi a többi szabályt
string Szöveges érték
integer Egész szám
min:5 Minimum érték (számoknál) / minimum hossz (szövegnél)
max:100 Maximum érték / hossz
unique:tabla,mezo Egyedi az adott táblában
boolean Igaz/Hamis (1/0)
email Érvényes email formátum
date Érvényes dátum

12 Named Routes & Flash üzenetek

Named Routes (elnevezett útvonalak)

Ha a route-hoz adsz egy ->name()-t, akkor a Blade-ben és a Controllerben szimbolikusan hivatkozhatsz rá — ha az URL változik, nem kell mindent átírni.

// routes/web.php:
Route::get('/', [TCtrl::class, 'index'])->name('termekek.index');
Route::get('/create', [TCtrl::class, 'create'])->name('termekek.create');
Route::post('/store', [TCtrl::class, 'store'])->name('termekek.store');

// Controllerben - vissza a named route-ra:
return redirect()->route('termekek.index');

// Blade-ben link generálás:
<a href="{{ route('termekek.create') }}">Új terem</a>

Flash üzenetek (sikeres/sikertelen visszajelzés)

A ->with('kulcs', 'uzenet') egy egyszer látható session üzenetet küld. A Blade-ben így jelenítheted meg:

// Controllerben mentés után:
return redirect()->route('termekek.index')
    ->with('success', 'Terem sikeresen mentve!');

// Controllerben hiba esetén:
return redirect()->back()
    ->with('error', 'Valami hiba történt!');
{{-- Blade-ben a nézeten belül (pl. lista.blade.php): --}}
@if(session('success'))
    <div class="alert-success">{{ session('success') }}</div>
@endif

@if(session('error'))
    <div class="alert-error">{{ session('error') }}</div>
@endif

{{-- Validációs hibák megjelenítése egy mezőnél: --}}
@error('tanterem')
    <span class="text-red-500">{{ $message }}</span>
@enderror

13 Method Spoofing — PUT & DELETE HTML formokon

A böngésző HTML formokban csak GET és POST metódust tud küldeni. Ha a route Route::put() vagy Route::delete()-t használ, a form metódust "spoofolni" kell a @method() direktívával.

PUT (módosításhoz) — routes/web.php-ban:

Route::put('/update/{id}', [Ctrl::class, 'update'])->name('update');
// VAGY egyszerűbb megoldás vizsgán (mindkettő elfogadott):
Route::post('/update/{id}', [Ctrl::class, 'update']);

Form a Blade-ben (PUT-hoz @method kell): 

<form action="/update/{{ $telek->id }}" method="POST">
    @csrf
    @method('PUT')  {{-- ezzel jelezzük, hogy valójában PUT --}}

    <input type="text" name="cim" value="{{ $telek->cim }}">
    <button type="submit">Mentés</button>
</form>

DELETE (törléshez) — routes/web.php-ban:

Route::delete('/destroy/{id}', [Ctrl::class, 'destroy'])->name('destroy');
// VAGY — vizsgafeladatnál a GET is teljesen elfogadott:
Route::get('/delete/{id}', [Ctrl::class, 'destroy']);

DELETE form Blade-ben (ha nem GET): 

<form action="/destroy/{{ $adat->id }}" method="POST">
    @csrf
    @method('DELETE')  {{-- ezzel jelezzük a DELETE metódust --}}
    <button type="submit">Törlés</button>
</form>

{{-- EGYSZERŰBB megoldás vizsgán (ha GET van a route-ban): --}}
<a href="/delete/{{ $adat->id }}">Törlés</a>

14 Modell — Haladó beállítások & Lekérdezési tippek

$table, $casts, $timestamps

Ha a tábla neve nem egyezik a Laravel konvencióval (pl. Termek → termeks helyett tanterem), kézzel kell megadni:

class Termek extends Model
{
    // Ha a tábla neve NEM az, amit Laravel generálna:
    protected $table = 'tanterem';

    // Időbélyegek kikapcsolása:
    public $timestamps = false;

    // Automatikus típuskonverzió (cast):
    // -> a Blade-ben $adat->projektor már PHP bool lesz!
    protected $casts = [
        'projektor' => 'boolean',
        'tv'        => 'boolean',
    ];

    protected $fillable = ['tanterem', 'projektor', 'tv'];
}

select() — Csak bizonyos oszlopok lekérése

Ha a feladat csak bizonyos adatokat kér megjeleníteni: 

// Csak meghatározott oszlopok:
$adatok = Termek::select('tanterem', 'befogadokepesseg', 'szamitogepek_szama')->get();

// Feltétel alapján szűrés:
$olcsok = Auto::where('ar', '<', 5000000)->get();

// Több feltétel:
$szurt = Auto::where('ar', '<', 5000000)
             ->where('evjarat', '>', 2015)
             ->orderBy('ar')
             ->get();

// Count - megszámlálás:
$db = Termek::count();
$db2 = Termek::where('projektor', true)->count();

Manuális mentés vs create()

Mindkét módszer elfogadott vizsgán:

// 1. Módszer — MANUÁLIS (nem kell $fillable!):
$uj = new Telek();
$uj->cim = $request->cim;
$uj->ar  = $request->ar;
$uj->save();

// 2. Módszer — create() ($fillable szükséges a modellben!):
Telek::create($request->all());

// 3. Módszer — create() validált adatokkal:
Telek::create($request->validate([...]));

public function run(): void {
    $file = database_path('seeders/konyvek.csv');
    $handle = fopen($file, 'r');
    $headers = fgetcsv($handle, 0, ';');
    $headers = array_map(fn($h) => trim($h, '"'), $headers);

            'oldalszam' => (int) $data['oldalszam'],
            'elerheto'  => (bool) $data['elerheto'],
        ]);
    }
    fclose($handle);
}

    public function store(Request $request) {
        $konyv = new Konyv();
        $konyv->cim = $request->cim;
        $konyv->szerzo = $request->szerzo;
        $konyv->ev = $request->ev;
        $konyv->oldalszam = $request->oldalszam;
        $konyv->elerheto = $request->elerheto;
        $konyv->save();
        return redirect('/konyvek');
    }

    public function edit($id) {
        $konyv = Konyv::findOrFail($id);
        return view('edit', compact('konyv'));
    }

    public function update(Request $request, $id) {
        $konyv = Konyv::findOrFail($id);
        $konyv->cim = $request->cim;
        $konyv->szerzo = $request->szerzo;
        $konyv->ev = $request->ev;
        $konyv->oldalszam = $request->oldalszam;
        $konyv->elerheto = $request->elerheto;
        $konyv->save();
        return redirect('/konyvek');
    }

    public function delete($id) {
        $konyv = Konyv::find($id);
        if ($konyv) {
            $konyv->delete();
            return redirect('/konyvek');
        }
        return redirect('/adatok');
    }
}

16 Vizsga leadás: Export & ZIP

1. Adatbázis Exportálás (SQL)

Nyiss egy új parancssort a projekt mappában, majd futtasd: 

mysqldump -u root -p adatbazis_neve > mentes.sql
  • -u root: felhasználónév
  • -p: jelszót kér (ha XAMPP-on nincs jelszó, hagyd le a -p-t)
  • Ha a mysqldump parancs nem található, használd a teljes elérési utat (XAMPP):
C:\xampp\mysql\bin\mysqldump -u root adatbazis_neve > mentes.sql

2. Projekt Tömörítése (ZIP)

A leadandó fájl mérete miatt a vendor mappát KI KELL HAGYNI!

  1. Másold be az előbb elkészült mentes.sql fájlt a projekt gyökérmappájába.
  2. Kattints jobb gombbal a projekt mappájára (pl. backendHT).
  3. Ha van 7-Zip: Jobb klikk -> 7-Zip -> Hozzáadás archívumhoz. Jelöld ki a fájlokat úgy, hogy a vendor mappa NINCS kijelölve.
