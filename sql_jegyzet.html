01

Egyszerű szűrés – WHERE

Mikor? Ha feltétel alapján kell szűrni sorokat.
SELECT nev, tomeg, ar FROM keverek WHERE tomeg <> 20;
SELECT ev, cim FROM film WHERE nyert > 0;
SELECT nev, orszag FROM allomas WHERE orszag IS NOT NULL;

Fontos operátorok:

  • =, <>, >, <, >=, <=
  • IS NULL / IS NOT NULL – hiányzó értékre szűrés
  • AND, OR – feltételek kombinálása
  • LIKE "%szó%" – tartalmazza; LIKE "%szó" – végződik rá; LIKE "szó%" – kezdődik
  • BETWEEN x AND y – tartomány
02

Számított mező – SELECT-ben kifejezés

Mikor? Ha új értéket kell kiszámolni a meglévő oszlopokból.
SELECT ut, (kesz + epul + terv) AS osszeg FROM palya;
SELECT nev, ar * 1000 / tomeg AS Kilogrammonkent FROM keverek;
Az AS ad nevet a kiszámolt oszlopnak. Rendezésnél hivatkozhatsz a sorszámára is: ORDER BY 2 DESC.
03

Rendezés – ORDER BY

Mikor? Ha az eredményt sorba kell rendezni.
SELECT nev, tomeg FROM keverek ORDER BY tomeg DESC;      -- csökkenő
SELECT nev, ar FROM keverek ORDER BY ar ASC;               -- növekvő (alapértelmezett)
SELECT keverek.nev, osszetevo.nev FROM ... ORDER BY keverek.nev, osszetevo.nev; -- több szint
04

Duplikátumok eltávolítása – DISTINCT

Mikor? Ha ismétlődő értékek lehetnek, és minden értéket csak egyszer szeretnél látni.
SELECT DISTINCT rendezo FROM film WHERE ev > 2000;
SELECT DISTINCT ut FROM telepules WHERE nev IN (...);
05

Táblák összekapcsolása – JOIN

Mikor? Ha az adatok több táblában vannak, és össze kell őket fűzni.

Régi stílus (implicit JOIN)

SELECT film.cim, szinkron.szinesz
FROM film, szinkron
WHERE film.filmaz = szinkron.filmaz;

Új stílus (explicit JOIN)

SELECT film.cim, szinkron.szinesz
FROM film
JOIN szinkron ON film.filmaz = szinkron.filmaz;

Három tábla

SELECT keverek.nev, osszetevo.nev
FROM keverek, kapcsolat, osszetevo
WHERE keverek.id = kapcsolat.keverekid
  AND osszetevo.id = kapcsolat.osszetevoid;
Kapcsolótáblánál (pl. kapcsolat, vege, hely) mindig mindkét idegen kulcsot be kell kötni a WHERE-be.
06

Csoportosítás – GROUP BY

Mikor? Ha csoportonként kell valamit megszámolni, összegezni, max/min értéket keresni.
SELECT keverek.nev, COUNT(osszetevo.nev)
FROM keverek, kapcsolat, osszetevo
WHERE keverek.id = kapcsolat.keverekid
  AND osszetevo.id = kapcsolat.osszetevoid
GROUP BY keverek.nev;
FüggvényMit csinál
COUNT(*)sorok száma
COUNT(mezo)nem NULL értékek száma
SUM(mezo)összeg
MAX(mezo)legnagyobb érték
MIN(mezo)legkisebb érték
MAX(ev) - MIN(ev)évek terjedelme
A SELECT-ben csak olyan mező szerepelhet, ami vagy GROUP BY-ban van, vagy aggregált függvénybe van csomagolva.
07

Csoportszűrés – HAVING

Mikor? Ha a csoportosítás után kell szűrni (pl. "amelyikben több mint 1 rekord van").
WHERE

Sorok szűrése csoportosítás ELŐTT

HAVING

Csoportok szűrése csoportosítás UTÁN

SELECT ev FROM film GROUP BY ev HAVING COUNT(ev) >= 10;

SELECT keszito.nev, COUNT(cim)
FROM film, kapcsolat, keszito
WHERE film.id = kapcsolat.filmid AND keszito.id = kapcsolat.keszitoid
  AND producer > 0
GROUP BY keszito.nev
HAVING COUNT(cim) > 1;
08

Legjobb N elem – ORDER BY + LIMIT

Mikor? Ha csak a legtöbbet / legkevesebbet / legdrágábbat kell visszaadni.
-- Legtöbb összetevőjű keverék (1 db):
SELECT keverek.nev, COUNT(osszetevo.nev)
FROM keverek, kapcsolat, osszetevo
WHERE keverek.id = kapcsolat.keverekid AND osszetevo.id = kapcsolat.osszetevoid
GROUP BY keverek.nev
ORDER BY COUNT(osszetevo.nev) DESC
LIMIT 1;

-- 2. legtöbb ut-tal rendelkező település:
SELECT nev, COUNT(ut) FROM telepules GROUP BY nev ORDER BY 2 DESC LIMIT 1, 1;
--                                                                       ^  ^
--                                                               kihagyás  db
  • LIMIT 1 = csak az első sor
  • LIMIT 1, 1 = kihagyja az első sort, a következőt adja vissza (2. helyezett)
LIMIT 1 holtverseny esetén csak EGYET ad vissza. Ha mindegyiket kell → használj HAVING-et (ld. 10. pont).
09

Allekérdezés – IN / NOT IN

Mikor? Ha egy halmaz alapján kell szűrni: "amelynek id-ja szerepel egy másik lekérdezés eredményében".

IN – szerepel a halmazban

-- Azok a keverékek, amelyek tartalmaznak paradicsomot:
SELECT nev FROM keverek
WHERE id IN (
    SELECT keverekid FROM kapcsolat, osszetevo
    WHERE osszetevo.id = kapcsolat.osszetevoid AND osszetevo.nev = 'paradicsom'
);

NOT IN – NEM szerepel a halmazban

-- Azok a keverékek, amelyek NEM tartalmaznak borsot:
SELECT nev FROM keverek
WHERE id NOT IN (
    SELECT keverekid FROM kapcsolat, osszetevo
    WHERE osszetevo.id = kapcsolat.osszetevoid AND osszetevo.nev LIKE '%bors'
);
⚠️ Miért nem elég WHERE osszetevo.nev != '%bors'? Mert ha egy keveréknek van borsos ÉS nem borsos összetevője is, a nem borsos sor miatt a keverék megjelenik. A NOT IN az egész keveréket kizárja, ha bármelyik összetevője bors.

Mindkét feltétel teljesül – IN + IN

-- Paradicsomos ÉS chilis keverékek:
SELECT nev FROM keverek
WHERE id IN (SELECT keverekid FROM kapcsolat, osszetevo
             WHERE osszetevo.id = kapcsolat.osszetevoid AND osszetevo.nev = 'paradicsom')
  AND id IN (SELECT keverekid FROM kapcsolat, osszetevo
             WHERE osszetevo.id = kapcsolat.osszetevoid AND osszetevo.nev = 'chili');

Ugyanez HAVING-gel (alternatíva)

SELECT keverek.nev
FROM keverek, kapcsolat, osszetevo
WHERE keverek.id = kapcsolat.keverekid AND osszetevo.id = kapcsolat.osszetevoid
  AND osszetevo.nev IN ('paradicsom', 'chili')
GROUP BY keverek.nev
HAVING COUNT(osszetevo.id) = 2;
10

Skaláris allekérdezés WHERE-ben

Mikor? Ha egyetlen értékkel kell összehasonlítani, amit egy másik lekérdezés ad vissza.
-- Legtöbb összetevőt tartalmazó keverékek (holtverseny esetén mind):
SELECT keverek.nev, COUNT(osszetevo.nev)
FROM keverek, kapcsolat, osszetevo
WHERE keverek.id = kapcsolat.keverekid AND osszetevo.id = kapcsolat.osszetevoid
GROUP BY keverek.nev
HAVING COUNT(osszetevo.nev) = (
    SELECT MAX(db) FROM (
        SELECT COUNT(osszetevo.nev) AS db
        FROM keverek, kapcsolat, osszetevo
        WHERE keverek.id = kapcsolat.keverekid AND osszetevo.id = kapcsolat.osszetevoid
        GROUP BY keverek.nev
    ) AS al
);
11

Allekérdezés FROM-ban (virtuális tábla)

Mikor? Ha egy összetett eredményt tovább kell szűrni, de egyetlen lekérdezésben nem megy.
-- Vonalak kezdő- és végállomása:
SELECT indulasi.vonalid, indulasi.nev, veg.nev
FROM (
    SELECT nev, vonalid FROM allomas, hely
    WHERE allomas.id = allomasid AND tav = 0
) AS indulasi,
(
    SELECT nev, vonalid, tav FROM allomas, hely
    WHERE allomas.id = allomasid
) AS veg,
(
    SELECT vonalid, MAX(tav) AS maxtav FROM hely GROUP BY vonalid
) AS tulso
WHERE indulasi.vonalid = veg.vonalid
  AND veg.vonalid = tulso.vonalid
  AND veg.tav = tulso.maxtav;
A FROM-ban lévő allekérdezést mindig el kell nevezni: AS nev.
12

Önillesztés – ugyanaz a tábla kétszer

Mikor? Ha ugyanabban a táblában kell két különböző sort összehasonlítani (pl. "akik Pap Katival együtt szerepeltek").
SELECT s1.hang, film.cim
FROM film, szinkron s1, szinkron s2
WHERE film.filmaz = s1.filmaz
  AND s1.filmaz = s2.filmaz
  AND s2.hang = 'Pap Kati'
  AND s1.hang <> 'Pap Kati'
ORDER BY film.cim, s1.hang;
A tábla két példányát különböző aliasszal (s1, s2) kell ellátni.
13

Szöveges mezőn alapuló kapcsolat

Mikor? Ha nem id-n, hanem szöveges mezőn kell összekapcsolni (pl. rendező neve = színész neve).
-- Olyan rendezők, akik egyben színészek is:
SELECT DISTINCT rendezo AS "Szinesz-rendezo"
FROM film, szinkron
WHERE film.rendezo = szinkron.szinesz;
14

LIKE – mintaillesztés

LIKE "%bors"      -- végződik rá (MySQL: %)
LIKE "%indiai%"   -- tartalmazza
LIKE "E%5"        -- E-vel kezdődik, 5-re végződik
LIKE "_____"      -- pontosan 5 karakter (minden _ egy karakter)
Vizsgán figyelj: Access-feladatokban * a wildcard (LIKE "*bors"), MySQL-ben %.
15

LENGTH() – karakterszám szerinti szűrés

SELECT DISTINCT ut FROM europa WHERE LENGTH(eurout) = 3 AND eurout LIKE "E%5";
16

IS NULL – hiányzó értékre szűrés

SELECT nev, orszag FROM allomas WHERE orszag IS NOT NULL;
SELECT palya.ut FROM palya WHERE kesz IS NOT NULL;
Soha ne használj = NULL-t – az mindig hamis! Mindig IS NULL / IS NOT NULL.
17

YEAR() – dátumból év kinyerése

SELECT cim FROM film WHERE YEAR(bemutato) >= 1939 AND YEAR(bemutato) <= 1945;
SELECT cim FROM film WHERE nyert > 0 AND YEAR(bemutato) - ev > 10;
